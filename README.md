# Analysis-and-Design
**Course Reflection**

Throughout this course, I worked on developing a course advising system that could efficiently load, organize, search, and display course information from a file. The primary problem I was solving was how to store and retrieve course data in a way that allowed users to quickly view a sorted list of courses and find prerequisite information for individual courses. To accomplish this, I evaluated different data structures and ultimately implemented a binary search tree (BST) because it provided efficient insertion, searching, and in-order traversal for displaying courses in alphanumeric order.

I approached the problem by first analyzing the program requirements and determining which data structure would best support the required operations. Understanding data structures was essential because the choice of data structure directly affected the program's efficiency, organization, and scalability. I compared the strengths and weaknesses of vectors, hash tables, and binary search trees before implementing the BST. This process reinforced the importance of selecting the appropriate data structure based on the application's needs rather than simply choosing the easiest option to implement.

One of the biggest roadblocks I encountered was correctly reading and validating the course data from the input file while maintaining the relationships between courses and their prerequisites. I also had to ensure the binary search tree insertion and traversal logic worked correctly so that courses were displayed in sorted order. I overcame these challenges by breaking the project into smaller tasks, testing each function individually, using sample input files, and carefully debugging the program. Adding meaningful comments and following a step-by-step development process also helped me identify and resolve issues more efficiently.

Working on this project expanded my approach to software design by teaching me to think more about performance, modularity, and long-term maintainability before writing code. Instead of focusing only on making the program work, I learned to consider how different design decisions impact efficiency, readability, and future enhancements. Separating functionality into individual functions and classes made the code easier to test, debug, and modify.

This project also improved the way I write programs that are maintainable, readable, and adaptable. I learned the importance of using descriptive variable and function names, writing clear inline comments, organizing code into logical sections, and designing reusable functions that each perform a single responsibility. These practices make the code easier for others to understand and allow future updates with minimal changes. Overall, this course strengthened both my programming skills and my understanding of how thoughtful software design and appropriate data structure selection contribute to building efficient, reliable applications.
